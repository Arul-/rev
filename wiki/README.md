# Rev Wiki

This wiki is the project memory for Rev.

Future Codex `/goal` runs should read this folder before making design changes
and update it when they learn something useful.

## Pages

- [Architecture](architecture.md) — how Rev works internally.
- [Decisions](decisions.md) — product and engineering decisions.
- [Testing](testing.md) — how to test Rev and what scenarios matter.
- [Hackathon Notes](hackathon-notes.md) — demo story, track fit, and judging
  notes.
- [Codex Review Research](research-codex-review.md) — prior art from Codex
  review, adversarial review, rescue, and stop gates.

## Update Rule

Update the wiki when:
- the CLI behavior changes
- the report format changes
- a test scenario exposes a useful edge case
- a Codex `/goal` run reveals a limitation
- hackathon positioning changes

Do not paste large logs. Summarize the learning and link to files or commands.
